## 2026-04-05 安装流程修复、管理员恢复模式与文档同步

### 安装与创建管理员流程修复
- 修复数据库初始化完成后，`admin/setup.html` 在检测到已有管理员时会快速强制跳转到 `admin/login.html` 的问题。
- 调整首次创建管理员页面：检测到旧管理员数据时，页面改为停留并明确提示“当前数据库中已存在管理员账号”，不再直接强跳。
- 安装完成后的收尾跳转改为先回到 `setup.html` 确认状态，再由页面根据实际状态决定后续操作。

### 新增管理员恢复模式（默认关闭）
- `api/config.php` 新增 `ADMIN_RECOVERY_ENABLED` 与 `ADMIN_RECOVERY_KEY` 配置项，用于处理“旧数据库里残留管理员账号、但当前维护者无法确认原账号”的场景。
- `api/admin.php` 新增 `recovery_reset` 接口：仅在恢复模式启用且恢复密钥正确时，才允许清空 `admins` 表并重新进入首个管理员创建流程。
- 恢复操作增加多重保护：CSRF 校验、限流、确认文本 `RESET ADMINS`、错误日志记录、执行后自动清除当前管理员会话。
- `api/check_install.php` 新增恢复模式状态回传，供 `admin/setup.html` 显示“仅说明”或“恢复操作面板”。
- `api/install.php` 配置写入逻辑已同步支持恢复模式配置项，避免安装流程覆盖该设置。

### 文档与页面提示
- `admin/setup.html` 新增恢复说明卡片：明确告知用户应前往项目根目录 `/api/config.php` 手动配置恢复模式，并提示恢复完成后立即关闭开关。
- `README.md` 同步补充管理员恢复模式的使用场景、配置方法、执行步骤与安全说明。
- `LICENSE` 新增说明性前言：提示功能说明见 `README.md`，版本变更见 `CHANGELOG.md`，且不改变 MIT 授权条款本身。

## 2026-04-03 代码瘦身、模块化重构与集中修复

本次覆盖前端 JS 模块化拆分、后端 API 层瘦身、UI 图标统一化，以及针对历史功能的两轮 Debug 修复。

### 前端 JS 模块化拆分

将 `main.js`（88KB）与 `admin.js`（97KB）拆分为 6 个职责明确的模块，消除跨文件重复代码。

**新增模块：**
| 模块 | 体积 | 职责 |
|------|------|------|
| `common.js` | 6KB | `apiFetch`、CSRF、`escapeHtml`、Toast、剪贴板、分页渲染器、格式化工具 |
| `ui.js` | 3KB | 侧边栏、主题切换、页面路由、滚动监听 |
| `orders.js` | 12KB | 订单凭据解析、VPS 信息复制、退款弹窗、规格快照渲染 |
| `notifications.js` | 11KB | 通知面板、轮询、通知中心全页面 |

**消除的重复：**
- `escapeHtml` / `showToast` / `formatFileSize` — 两文件完全一致，提取至 `common.js`
- `initCsrfToken` / `apiFetch` — 统一为带 `apiBase` 参数的版本
- 三套分页逻辑合并为 `renderPaginationWidget()`
- 通知点击 / 全部已读合并共用跳转逻辑

**净效果：**
- `main.js` 88KB → 47KB（-46%）、`admin.js` 97KB → 73KB（-25%）
- `index.html` 内联脚本 94 行 → 0 行
- JS 总量 185KB → 152KB（-18%）

### 后端 API 层瘦身

消除 `api/` 与 `includes/` 中的冗余包装函数，不改变任何外部行为。

**删除冗余函数（11 个）：**
- `ordersHasColumn` / `tableExistsDash` / `updTableExists` / `updColumnExists` / `fillOrderCredentialsVisibility` / `fillAdminOrderOutput` / `notificationsEnabled` / `templateFeatureReady` / `productHasTemplateColumn` / `ticketsHasColumn` / 本地 `sendEmail` — 统一改用 `commerceTableExists` / `commerceColumnExists` / `commercePrepareOrderForOutput` / `sendSmtpEmail`

**新增公共工具（3 个）：**
- `paginateParams()` / `paginateResponse()` — 统一分页参数与响应构造，覆盖 6 个接口文件
- `sendSmtpEmail()` — 统一 SMTP 邮件发送，PHPMailer 优先、`mail()` 回退

**净效果：** 总行数 ~7800 → ~7550，净减约 250 行

### 前端 UI 优化

- **Emoji → SVG 图标统一**：后台侧边栏 15 个 emoji 全部替换为 16×16 线性 SVG；前台空状态图标（实例 / 商品 / 订单 / 工单 / 公告）替换为 48×48 SVG
- **未登录状态优化**：可用实例、新建实例、我的订单、我的工单四个页面在未登录时显示锁图标 + 登录引导，不再显示 loading 占位或空数据
- **性能修复**：`updateHomeStats` 去除重复的 `updateWelcomeCard` 调用；`loadCreditSummary` 改为局部更新余额显示，避免整体 DOM 重渲染

### 两轮 Debug 修复

针对 **2026-03-14 / 2026-03-15 新增功能** 做代码级排查，重点清理"功能已加上，但在旧库、缺字段、真实配置场景下会失败"的问题。

**第一轮：**
- 修复社区规则保存失败（误用不存在的 `reason` 字段 → 统一 `remark`）
- 修复信任等级折扣备注无法保存
- 修复社区页 / 积分页在旧库上直接 SQL 异常（改为按表按列动态降级）
- 修复交付信息覆盖交付备注（`delivery_info` 与 `delivery_note` 分离写入）

**第二轮：**
- 修复通知邮件 SMTP 配置"能填不能用"（优先 PHPMailer，失败回退 `mail()`）
- 修复余额摘要长期显示"最近变动：暂无"
- 修复余额调整 / 商品模板 / 社区规则在旧库缺字段或缺表时直接失败
- 修复商品新增编辑依赖新字段过强（扩展字段改为按列动态写入）
- 修复用户登录 / OAuth 登录在旧库下崩溃
- 修复支付请求映射状态残留

**校验：** 全部 PHP 通过 `php -l`，全部 JS 通过 `node --check`

## 2026-03-15 功能升级、重构与集中修复

这是一次体量较大的迭代，覆盖了前后台功能扩展、订单展示修复、退款流程完善、代码去重整理以及数据库兼容增强。

### 主要新增与增强
- 新增 **站内余额钱包**：支持余额汇总、余额流水、后台加减积分、余额变动通知。
- 新增 **余额支付能力**：保留优惠券链路，支持“优惠后余额支付”，同时继续兼容 EasyPay 外部支付。
- 新增 **商品模板系统**：支持模板绑定商品、快速回填规格信息。
- 新增 **Linux DO 社区规则能力**：支持最低信任等级、silenced 风控、白名单 / 黑名单、信任等级折扣。
- 新增 **工单增强**：支持分类、优先级、内部备注、核实状态、退款审核字段、事件时间线、回复模板、附件上传审计。
- 新增 **通知中心升级**：站内通知基础上扩展邮件 / Webhook 配置。
- 新增 **后台统计报表**：覆盖成交额、支付方式统计、热门商品、工单分类占比、余额总量、异常订单统计等。
- 新增 **退款方式扩展**：后台可选择退回站内余额或原路退回支付账户。
- 新增 **订单快照字段**：商品名称、规格、地区、线路、系统、连接信息等会在下单时保留快照，避免删除商品后历史订单信息丢失。

### 前后台体验与功能修复
- 修复前台缺少 `orderDetailModal` 节点、点击订单详情无法弹出的问题。
- 修复“可用实例”旧逻辑误跳转到订单列表的问题，明确区分“新建实例”和“可用实例”语义。
- 修复订单商品被删除后，历史订单显示“商品已删除”且规格全空的问题；查询时优先使用订单快照。
- 修复后台订单列表与订单详情错误读取用户名字段，导致订单用户显示为 `-` 的问题。
- 修复后台“积分管理 / 搜索用户”前端仍按旧数组结构读取，导致明明有用户却显示“暂无用户”的问题。
- 修复欢迎语和用户态偶发显示 `[object Object]`、用户名丢失等问题。
- 修复浏览器从支付页返回后前台读取旧缓存，导致“加载失败，请刷新重试”的问题。
- 修复商品支付成功后仍继续显示“在售”的问题。
- 修复订单支付状态与交付状态混淆，前后台可分别看到真实支付进度与交付进度。

### 交付与退款链路修复
- 修复数据库更新脚本遗漏 `orders.delivery_info` 字段的问题。
- 修复后台填写交付信息后仍保持“待开通”的问题；保存交付信息时自动切换交付状态。
- 修复后台“标记已交付”时 `delivery_info` 未实际写入订单表的问题。
- 修复前台订单详情无法正确识别并展示交付信息、异常说明、连接信息的问题。
- 修复订单详情中加密交付字段直接显示 `enc:...` 原文的问题，新增统一解密展示逻辑。
- 修复已退款 / 已取消订单仍可查看详情、复制连接信息、重复发起退款的问题。
- 修复退款事务缺少行锁校验导致同一订单可能被重复退款的严重漏洞。
- 修复后台退款后详情弹窗未关闭，导致用户可继续误点退款按钮的问题。
- 前台退款从简单 prompt 升级为正式表单弹窗，可填写退款原因、补充说明，并选择原路退回或退回站内余额。
- 后台退款工单详情增加审批区，可修改退款方式、填写审批备注后执行自动退款。

### OAuth、支付与兼容修复
- 修复 Linux DO OAuth 在 `state` 丢失或重复回调场景下提示“安全验证失败”，但返回首页后又已登录的体验问题；错误页增加重新登录入口。
- 修复 EasyPay 多次拉起同一未支付订单时，`out_trade_no` 冲突导致唯一键报错的问题；新增 `payment_requests` 映射表生成唯一外部请求号。
- 修复异步支付通知只按 `orders.order_no` 匹配，导致新外部支付请求号无法正确回写订单的问题。
- 修复旧站升级时，部分新增字段重复创建被误判为失败的问题。
- 修复旧库缺少 `product_templates` 或商品扩展字段时，商品列表接口直接报错导致前台“加载失败”的问题。

### 代码整理
- 重构 `js/main.js`：合并重复定义函数，删除历史补丁块，按模块分区整理。
- 整理 `js/admin.js`：合并重复函数定义，按模块梳理后台逻辑。
- 清理 `index.html`、`api/*.php`、`includes/*.php` 中的冗余与重复逻辑。
- 统一前台静态资源版本号，降低缓存导致旧脚本继续生效的概率。

### 数据库与升级说明
- `orders` 增加商品快照、连接信息、退款相关与交付相关字段。
- `users` 增加余额、Linux DO 状态、更新时间、邮箱校验等扩展字段。
- `products` 增加模板、地区、线路、系统、信任等级、审核限制、白名单限制、排序等字段。
- 若旧订单创建于快照字段上线前，且对应商品已被物理删除，则历史规格无法完全反推；本次修复后新订单会持续保留快照。

## 2026-03-14 安装兼容与依赖兜底

### 核心修复
- 修复安装器写入 `api/config.php` 后，启用 OPcache 时仍可能读取旧配置的问题。

### 校验
- 全部 PHP 文件通过 `php -l`
- 全部 JS 文件通过 `node --check`
- 本地已验证安装器生成密钥后，`has_encryption_key` 状态可正常返回
